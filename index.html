Notes

Enter your competitors' name, service area cluster number, container size, and monthly rate.

The Cluster # must match a cluster ID from the map clustering run. These prices will be used as the Price Match floor when calculating New Rates.

If a customer's current price already exceeds the competitor price, no increase is applied (hold pricing).

Min Base Price

Cont Size (yd) Min Base Price ($) Price / Yard

Discount Parameters

This should be the lowest price you would accept. If a competitor's price is below your minimum, this rate overrides the price match.

Preferred Price

Cont Size (yd) Preferred Price ($) Price / Yard

Notes

These rates should be your average target prices to help prevent price erosion.

The Preferred Price is adjusted upward for isolated accounts (those beyond Epsilon miles from any neighbor) using the Extra $/Mile driver.

Cluster Parameters

Epsilon (miles)Maximum radius between two points for one to be considered a neighbor of the other. Start with 1–5 miles for dense urban data, 10–25 for rural.
Min PointsMinimum number of points required to form a cluster. Points that don't meet this threshold are marked as noise (shown in black on the map).

Pricing Driver Parameters

ORS API KeyFree key from openrouteservice.org. When provided, nearest neighbor is found by drive time rather than straight-line distance. Leave blank to use straight-line miles.
Qty DiscountDiscount rate applied to the Preferred Price NN Adjusted when a service line has Qty > 1 (e.g. 0.3 = 30% off preferred price).
Extra $/MileDollar amount added per mile beyond Epsilon for accounts isolated from the cluster. Accounts within Epsilon get no extra charge.
PI RateTarget price increase rate applied to the current price (e.g. 0.1 = 10%). The New Rate will be at least Current Price × (1 + PI Rate), floored by Min Base Price.
Pref BufferInformational buffer on the Preferred Price (stored in state, displayed on Dashboard). Does not currently affect New Rate calculation.
